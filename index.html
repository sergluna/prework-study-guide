HTML

  • The body element represents the visible content shown to the user.

CSS

  • When applying a CSS rule, "*" means the rule will be applied to all the elements visble on the page
  • div {color: blue; text-align: justify;}
  • div is the selector which defines the element or attributes to which the rules, or declarations, will apply.
  • Declarations contain two important components proprety:value, the CSS property we want to apply and the value of the property. For example, the declaration color: blue; includes the CSS property color and the value for that property, blue.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The period preceding card designates this as a class selector. Any HTML element assigned to the card class will gain these property assignments from the CSS.
  • A link element creates a connection to an external file
  • The rel attribute, which goes inside of the link brackets, specifies the relationship between the current document and the linked document resource.
  • The href attribute, also goes inside of link brackets, specifies the location (URL) of the external resource. The href attribute is using a relative path. This means the location is derived from the current location (in this case, index.html). To locate the style sheet from the current location, we must look in the assets folder, hence assets/style.css.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git checkout: switch to another branch and check it out into your working directory
  • To navigate to your home directory, use "cd" or "cd ~"
  • To navigate to the previous directory (or back), use "cd -"
  • To navigate up one directory level, use "cd .."
  • git branch: list your branches, a * will appear next the currently active branch
  • Git Flow: git add -A, git commit -m "add a message of changes", git pull origin main, git push origin feature/"current branch"
  • git add -A: Adds the changes files in our working directory to the stagging area of Git
  • git commit -m "": commits our changes to out repo
  • git commit origin main: let's us make sure our current branch is up to date before making a pull request
  • git push origin feature/"current branch": pushes up all our changes
  • git pull origin main: pulls the remote branch into the local main branch (be sure to be in main branch in Git Bash)
